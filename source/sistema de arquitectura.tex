\section{Implementación}

Esta sección describe la materialización técnica del diseño propuesto en un entorno de prueba sobre Stellar testnet.

\subsection{Implementación de contratos Soroban}

Se desarrollan los contratos principales para representar tickets, gestionar reventa y controlar redención.

\subsubsection{NFT Ticket (OpenZeppelin)}

Se adapta un patrón NFT para emisión y gestión de propiedad de tickets con identificador único.

\subsubsection{Marketplace y pagos (XLM + USDC\_SIM)}

La lógica de mercado implementa listado, compra y liquidación con distribución de comisiones usando activos de prueba.

\subsubsection{Redeem y control de acceso}

La verificación en puerta combina validación de estado del ticket y control de titularidad para evitar reutilización.

\subsection{Backend Node.js + Stellar SDK}

El backend centraliza integración contractual, reglas de negocio y exposición de servicios para la interfaz web.

\subsubsection{API (usuarios, eventos, listings, verificación)}

Se implementan endpoints para autenticación, consulta de entradas, operaciones de reventa y validación.

\subsubsection{Indexer (ingesta de eventos)}

Un proceso dedicado consume eventos on-chain y los almacena para trazabilidad histórica.

\subsubsection{Persistencia Postgres (historial, listados, auditoría)}

La base de datos relacional conserva entidades de negocio, historial de transacciones y evidencias para análisis.

\subsection{Frontend web (simulación ticketera)}

La interfaz permite simular flujos B2B2C de compra, listado, reventa y verificación de tickets.

\subsection{Despliegue distribuido (3 nodos)}

El sistema se despliega en nodos diferenciados para capa de presentación, capa de servicios y capa de datos/indexación.

\subsection{Pruebas (unitarias, integración, antifraude, medición)}

La validación técnica incluye pruebas unitarias de contratos, integración entre servicios, escenarios de fraude y medición de desempeño.
